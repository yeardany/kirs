Git学习笔记

Sep 6, 2016 | essay | Hits

Git是一个开源的分布式版本控制系统，可以有效、高速的处理从很小到非常大的项目版本管理。git是Linus Torvalds为了帮助管理Linux内核开发而开发的一个开放源码的版本控制软件。Git是一款免费、开源的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目（百度百科），以下是我的git常用操作命令记录📝

一般步骤

  • 项目文件：首先有一个项目文件夹，切换到项目文件夹下面运行 git init 使它变成一个仓库
  • 暂存代码：每次编辑完项目后运行 git add . 使git将改动的代码提交到缓存区
  • 改动说明：提交代码前，运行 git commit -m “本次提交的说明”

注意：git add命令实际上就是把要提交的所有修改放到暂存区，然后，执行git commit就可以一次性把暂存区的所有修改提交到分支，记得所有文件修改完后先执行git add最后再执行git commit

  • 远程仓库：本地与远程仓库的链接，运行 git remote add origin git@git.coding.net:yeardany/works.git

注意：远程仓库名称就为origin这是Git默认的叫法，当然也可以改成别的，但是origin这个名字一看就知道是远程库。另，项目git仓库地址改成自己的，SSH免密码上传在此不再赘述

  • 远程推送：如果第一次推送到远程仓库 git push -u origin master ，之后推送如果远程有与本地相同的文件，先 git pull 一下后再直接git push

其他步骤

  • 删除与远程仓库关联 git remote remove origin
  • 直接丢弃工作区的上一次修改时，用命令 git checkout -- file
  • 删除文件 先删除暂存取文件 git rm test.txt 再删除本地文件 rm 'test.txt'

总结一波

第一次配置完成后只需要记住三个命令：git add . git commit -m "" git push
