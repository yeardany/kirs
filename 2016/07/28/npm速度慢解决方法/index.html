Npm速度慢解决方法

Jul 28, 2016 | javascript | Hits

说起node就会用到npm，无奈，由于GFW的原因，往往下载速度奇慢…项目安装个别模块还可以手动设置代理：npm install express --registry "http://registry.npm.taobao.org",但是一碰到yeoman构建项目时自动下载大量组件时就行不通了，之后发现nrm可以解决此类问题，使用它可以方便的切换代理

简介

nrm 是一个 NPM 源管理器，允许你快速地在如下 NPM 源间切换：

  • npm
  • cnpm
  • strongloop
  • european
  • australia
  • nodejitsu
  • taobao

安装

首先用npm安装：npm install -g nrm ，如果不行，
可以：npm install nrm -g --registry "http://registry.npm.taobao.org"

列出所有可供选择的源

nrm ls

npm —- https://registry.npmjs.org/
cnpm — http://r.cnpmjs.org/
taobao - http://registry.npm.taobao.org/
eu —– http://registry.npmjs.eu/
au —– http://registry.npmjs.org.au/
sl —– http://npm.strongloop.com/
nj —– https://registry.nodejitsu.com/ 带星号表示当前选择的源*

切换源

nrm use taobao

Registry has been set to: http://registry.npm.taobao.org/

测试每个源的速度

nrm test

npm —- 891ms
cnpm — 1213ms
taobao - 460ms
eu —– 3859ms
au —– 1073ms
sl —– 4150ms
nj —– 8008ms

其他

  1. 增加源：nrm add <registry> <url> [home]
  2. 删除源：nrm del <registry>
