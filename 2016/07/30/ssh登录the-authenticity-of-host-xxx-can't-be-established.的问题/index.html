SSH登录The Authenticity of Host Xxx Can't Be established.的问题

Jul 30, 2016 | linux | Hits

之前都用Xshell，介于mac终端直接支持Shell，就尝试直接ssh登录，运行ssh -p 22 <用户名>@<ip>发现一个奇怪的问题

The authenticity of host ‘< ip > (< ip >)’ can’t be established.
ECDSA key fingerprint is SHA256:….
Are you sure you want to continue connecting (yes/no)? yes
Host key verification failed.

然后，就没有然后了…😨

参照Xshell，会发现ssh登录会少了一个主机秘钥确认的步骤，猜想可能是第一次登录还未接受主机hostkey的关系，解决办法如下：执行ssh -o StrictHostKeyChecking=no <ip>或者把StrictHostKeyChecking no加到/etc/ssh/sshconfig 文件中，可以让ssh客户端自动接受新主机的hostkey,不用每次都自己输入yes。确认后重复登录步骤，根据提示输入远程主机密码即可😄
