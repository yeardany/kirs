组合 说明
A,B 匹配满足A（和/或）B的任意元素
A B 匹配满足B是A的后代结点（B是A的子节点，或是A子节点的子节点）
A>B 匹配满足B是A的直接子节点
A+B 匹配满足B是A的下一个兄弟节点（AB有相同的父结点且B紧跟在A之后）
A~B 匹配满足B是A之后的兄弟节点中的任意一个（AB有相同的父节点，B在A之后，但不一定是紧挨着A）

属性选择器

  • 存在和值属性选择器
    • [attr]选择包含attr属性的所有元素
    • [attr=val]选择attr属性被赋值为 val 的所有元素
    • [attr~=val]选择attr属性的值（以空格间隔出多个值）中有包含 val 值的所有元素，比如位于被空格分隔的多个类（class）中的一个类
  • 子串值属性选择器
    • [attr|=val]选择attr属性的值以val或val-开头的元素
    • [attr^=val]选择attr属性的值以val开头的元素
    • [attr$=val]选择attr属性的值以val结尾的元素
    • [attr*=val]选择attr属性的值包含val的元素

样式权重

样式权重值

  • 行内样式 1000
  • id选择器 100
  • 属性选择器、类选择器、伪类选择器 10
  • 元素选择器、伪元素 1
  • 通配符 0

生效规则

  • 选择器不同但指向同一元素，权重不同，权重值高的生效
  • 选择器相同或权重相等，就近原则，靠近目标的生效
  • 内联样式和外联样式，就近原则，靠近目标的生效
  • 选中非目标元素时，就近原则，靠近目标的生效

模块化

使用行内样式的缺点

  • 样式不能复用
  • 样式权重太高，不容易覆盖
  • 表现层与结构层没有分离
  • 不能进行缓存，影响加载效率

使用导入样式的缺点

  • 导入样式只能放在style标签的第一行，放在其他行则无效
  • @import声明的样式不能充分利用浏览器的并发请求资源行为，其加载行为会咽喉触发或被其他资源加载挂起
  • 由于@import样式表延后加载，会导致页面闪烁

性能优化

@import

避免使用@import：使用@import引入css文件只有在引用它的那个css文件被解析完成之后，浏览器才会知道还有另外一个css需要下载，导致浏览器无法并行下载；多个@import会导致下载顺序紊乱

重排

以下会导致重排

  • 改变窗口大小
  • 改变文字大小
  • 添加删除样式表
  • 内容改变，输入框输入
  • 伪类激活
  • 操作class属性
  • 操作dom
  • 设置style属性
  • 改变元素的内外边距
  • css内计算表达式

重排元素

  1. 大小有关的 width,height,padding,margin,border-width,border,min-height
  2. 布局有关的 display,top,position,float,left,right,bottom
  3. 字体有关的 font-size,text-align,font-weight,font-family,line-height,white-space,vertical-align
  4. 隐藏有关的 overflow,overflow-x,overflow-y

避免重排

  • 减少修改dom
  • 为动画元素使用绝对定位
  • 不使用table布局
  • 避免设置大量行内样式，使用class

重绘

元素外观改变，但布局没变，常见的重绘

  • 颜色 color,background
  • 边框样式 border-style,outline-color,outline,outline-style,border-radius,box-shadow,outline-width
  • 背景有关 background,backgound-image,background-position,background-repeat,background-size

其他

  • 文件压缩
  • 去除无用css
  • css匹配从右向左，因此不要使用嵌套过多过于复杂的选择器
  • 通配符和属性选择器效率最低，需要匹配的元素最多，尽量避免使用
